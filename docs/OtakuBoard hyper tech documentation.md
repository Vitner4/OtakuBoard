# Что такое OtakuBoard 
- **OtakuBoard** - это Open Source, Web-приложение для коллекционирования различного цифрового контента в виде "контент-блоков" (Визуальное представление контента в приложении) на сайте и JSON файлах локально на ПК. Приложение даёт возможность создать группу (Например: Аниме, персонажи, музыка, фильмы, книги и другое) и хранить в ней определенный контент, фильтровать его, создавать личные топы и рейтинги контента. В самих "контент-блоках" возможно указать название контента, год, автора, описание, ссылки, жанры, оценку, место в личном топе, теги и группу. 

- Посмотрели аниме или какой-нибудь хороший фильм? Так сохраните его в виде небольшого достижения и создайте собственную коллекцию цифрового контента, в приложении OtakuBoard!
# Цель
- Создать простое и удобное десктопное Web-приложение, для хранения цифрового контента в файлах локально на устройстве пользователя c ОС Windows/Linux. 

# Задачи
- **Простой и удобный Web-UI** c возможностью открыть его в отдельном окне или в основном браузере пользователя.
- **Регистрация в приложении и проверка на наличие пользователя**, создание локального репозитория пользователя.
- **Создание новых "контент-блоков"** и сохранение их в локальных файлах в локальном репозитории пользователя.
- **Алгоритм вывода локальных файлов** из локального репозитория пользователя и создание на основе их "контент-блоков".
- **Создание групп цифрового контента**.
- **Фильтрация и поиск цифрового контента**.
- **Страница аккаунта пользователя и настройки приложения**.
- **Страница авторов и ссылка на GitHub проекта OtakuBoard**.
- **Сборка приложения под ОС Windows/Linux**.

# Реализация
## 1. Frontend
### Общие элементы окон
(Элементы которые должны быть в каждом окне)
- **Стиль** - Соблюдать похожие приятные цвета, использовать мягкие шрифты и приятную стилистику. 
- **Шапка** - Динамичная, полупрозрачная. в начале шапки логотип приложения как кнопка, ведущая на главный экран, следом название страницы, вкладка "группы", вкладка "создать". в конце кнопка аккаунта пользователя в виде круга с фото.
- **Подвал** -  Указать год js функцией, копирайт, название приложения с ссылкой на GitHub проекта, об авторах с ссылкой на страницу авторов.
### Начальное окно (index.html)
- Окно, которое будет появляться при самом первом входе в приложение или при отсутствии аккаунта. Экран поделён на две части. Слева приветственные слова и кнопки входа и регистрации аккаунта, справа однотонный фон или картинка (на усмотрение)
### Окно входа и выбора аккаунта (authorization.html)
- Дизайн как у "Начальное окно", заголовок "Вход", Кнопка "Открыть аккаунт" для поиска папки аккаунта на устройстве.
### Окно регистрации аккаунта (registration.html)
- Дизайн как у "Начальное окно". заголовок "Регистрация", поле "Ваше имя", поле "Ваш псевдоним" который будет как id и который нельзя поменять, сделать уточнение об отсутствии изменения, кнопка "Создать аккаунт".
### Главное окно (listPage.html)
- Окно где будет располагаться все "контент-блоки". первый блок серый с прозрачным плюсом для создания нового блока, далее располагаются другие блоки (создать примерный вид блока для дальнейшего создания новых на основе его, с помощью скрипта). блок для фильтра и поиска "контент-блока" по названию.
### Окно создания нового блока (newBlock.html)
- Заголовок: создание нового контент-блока. Блок для выбора фотографии (выбор локального открытия или Web-ссылки). В конце кнопка "Создать".
- Поля для заполнения:
	- Имя контента
	- Тип контента (Аниме, Фильм, Манга)
	- Год создания
	- Жанр контента
	- Автор Контента
	- Компания производителя
	- Мировой рейтинг
	- Личный рейтинг (Цифровой от 0 до 10)
	- Личный статус аниме (Легендарное, Любимое...)
	- Ссылка на источник для просмотра
### Окно просмотра блока (blockView.html)
- Окно со всеми теми элементами и полями что были при создания нового блока ("Окно создания нового блока"), но с возможностью редактирования и сохранения.
### Пользовательское окно (userPage.html)
- Окно с фотографией пользователя, именем и псевдонимом с возможностью поменять фото или имя пользователя. Раздел с настройками (поменять тему, открытие в отдельном окне или в браузере...) и возможностью выйти из аккаунта.
### Авторское окно (autorPage.html)
- Hub с ссылками на авторов и их позицию в проекте (backend \ frontend разработчик)
